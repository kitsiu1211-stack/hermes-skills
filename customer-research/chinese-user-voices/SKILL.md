# 中文"听用户声音"市场验证方法论

中国市场没有Reddit，但有多个分散的阵地可以听到真实用户声音。本skill提供系统化的搜索策略和已验证的挖法。

## 适用场景

- 用户提出一个产品/机会想法（如农业AI、SaaS工具等），需要验证真实用户是否有这个痛点
- 需要了解某个行业的用户真实抱怨、求助、需求表达
- 做竞品分析时想看到用户的原始反馈

## 阵地地图

| 阵地 | 适合听什么 | 怎么搜 |
|------|-----------|--------|
| **百度贴吧**（农业吧/种植吧/养殖吧等） | 农户真实需求、求助、踩坑 | 搜"求助+病虫害"、"不知道+什么病" |
| **抖音评论区** | 最底层的用户反馈 | 关注农技类/行业类大V视频评论区 |
| **头条微头条/头条搜索** | 农户碎片化表达 | 搜"卖粮亏了"、"错过价格"等情绪词 |
| **小红书** | 新农人/返乡青年痛点 | 搜"种植踩坑"、"病虫害求助" |
| **惠农网农技学堂** | 结构化农技问答 | 引擎索引能到，但页面可能404 |
| **拼多多农资评论区** | 用药/用肥真实反馈 | 搜"不管用"、"还是坏了" |
| **微信公众号评论区** | 行业大V文章下的高赞评论 | 找到头部账号看 |

## 搜索策略

### 第一步：选定搜索引擎

目前已验证可稳定使用的中文搜索渠道：
- **头条搜索**（so.toutiao.com）— 主力搜索引擎，百度/Bing/Google/知乎/Reddit均遭反爬拦截
- **微信公众号直访** — 特定公众号文章可直达
- 如果上述都不行，尝试用工具`web_search`的site:搜索

### 第二步：关键词设计

**核心技巧**：不要搜产品名（如"病虫害识别App"），搜**用户会用的表达**：

| 你想验证的痛点 | 搜索关键词 |
|---------------|-----------|
| 病虫害识别需求 | "求助+病虫害"、"不知道什么病"、"用什么药"、"作物病害对照表" |
| 价格行情需求 | "卖粮亏了"、"错过价格"、"什么时候卖最贵" |
| 知识问答需求 | "蔬菜发生病害"、"不知道什么病害别乱打药" |
| 通用的痛点 | "要是有一个App就好了"、"有没有工具能xxx" |

### 第三步：判断信号强弱

**强信号**（痛点真实存在）：
- 同一主题的内容**跨年度持续出现**（痛点长期未解决）
- 评论区活跃（农业类>20条评论算高）
- 有用户配图求助但无人专业解答

**中信号**（可能有需求）：
- 文章/视频播放量高但评论区少
- 行业资讯类而非用户原始表达

**弱信号**（谨慎验证）：
- 只有内容创作者在说，没有用户反馈
- 搜索命中的是广告/软文

## 验证案例：农业AI

实际验证结果（2026年5月）：

### ✅ 病虫害识别：需求真实且高频
- 头条搜"作物真菌细菌病毒病害对照表"→ 102条评论，讲"青枯病当真菌病连打3遍杀菌剂，钱花了"
- 贴吧农业吧：用户配图求助西红柿小黑虫→ 回帖全卖肥料的广告，没有专业回答
- 2019年的内容2026年还能搜到→ **跨年度的长期未满足需求**

### ✅ 价格行情：农户普遍"卖亏了"
- "6月卖粮别盲目"→ 讨论"错过了合适售卖节点"
- "农户单枪匹马不知道价格行情"→ 无力感的直接表达

### ✅ 种植知识问答
- "蔬菜发生病害"→ 12万播放，102评论
- 说明"怎么判断+怎么用药"的内容有持续关注度

## 关键提醒

1. **找到需求 ≠ 有付费意愿**——还需要进一步验证。建议下一步：蹲行业微信群/抖音评论区直接问"如果有个App拍照就能xxx，你愿意用吗？"
2. **注意平台用户偏差**：贴吧/头条偏传统农户（需要微信就能用+打字少+发图就行的工具），小红书偏年轻新农人（愿意用新工具但挑剔）
3. **最关键的验证方法不是搜，是"蹲"**：连续盯某一个阵地3-7天，看有没有人主动问"有没有什么App能xxx"

## 步骤

1. 明确要验证的痛点和目标用户群
2. 从阵地地图中选2-3个最相关的平台
3. 设计"用户会用的表达"作为搜索关键词
4. 头条搜索（so.toutiao.com）优先尝试
5. 返回每个平台的实际搜索结果+信号判断
6. 汇总结论：痛点真实存在/存疑/不成立
7. 如验证通过，给出下一步验证（付费意愿/成交路径）的建议
