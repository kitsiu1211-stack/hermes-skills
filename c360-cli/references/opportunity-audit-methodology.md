# AI 商机盘点方法论

用于跨 CSM 团队（南区五线）的 AI 商机盘点。核心原则来自 2026.7.22 袁鑫杰的纠正。

## 核心规则

**新商机 = 本季度明确有增购或新购 AI 包的客户。已购客户不计入。**

## 判断流程

1. **获取各 RM 的 Q3 商机清单**（来自盘点会议妙记/字幕）
2. **交叉验证已购客户**：对照此前"已购 9.9+ 客户购买动因统计"等分析，剔除已购
3. **看会议原话中的关键信号**：
   - "Q3 推" / "本季度在推" → 新商机
   - "首购" / "新购" → 新商机
   - "增购" → 新商机
   - "已买" / "已XX万" / "已购" → 已购，非商机
   - "想推但缺场景" → 新商机（低确定性）
4. **按金额档位分桶**：9.9 / 29.7 / 49.7 / 99 / 待定
5. **按确定性分层**：🟢高 / 🟡中 / 🔴需更多信息

## 常见陷阱

| 陷阱 | 示例 | 正确处理 |
|------|------|---------|
| 表里写"已99"当商机 | 拓竹已99 → 放 Q3 商机列 | 移入附录"已购情况" |
| 已购但消耗好当增购 | 富途 29.7→99，历史路径 | 已购，除非有新购/增购动作 |
| 随飞书捆绑当 AI 商机 | 大疆（随飞书上线捆绑） | 不是 AI 商机，是套件捆绑 |
| 买过但未消耗当新购 | 领鑫 9.9 已买，有增购潜力 | 现有金额是已购，增购才算新商机 |

## 输出结构

1. 行业特征速览表（只列新商机）
2. 商机金额分布统计
3. 跨行业共性发现（insight）
4. 行业差异矩阵
5. Solid 商机拆解（按确定性）
6. 待办行动方案
7. 附录：已购情况

## 数据源优先级

1. 盘点会议妙记/字幕（最权威，有原话）
2. 五行业交叉分析表（用户手写汇总）
3. C360 CLI（订单/商机系统数据）
4. 早前"已购客户购买动因统计"
