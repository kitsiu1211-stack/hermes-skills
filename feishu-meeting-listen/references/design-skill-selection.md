# 设计技能选用铁律

## 核心教训

用户拥有多个设计 Skill（baoyu-design、apple-design、emil-design-eng 等），**禁止混用**。

**错误模式**（本 session 踩坑）：
- 拿 Zara 暖纸风格 + baoyu-design deck-stage → 混合后白字叠浅底，完全看不清 → 用户："好丑啊现在"
- 拿 baoyu-design 框架 + apple-design 细节尝试融合 → 看不出区别 → 用户："看着跟那个 baoyu skill 没有什么区别"

**正确做法**：
1. 用户说「用 XX skill 做」→ 只用那个 skill，不混入其他
2. 用户没指定 skill → 问一句「用 baoyu-design 还是 apple-design？」
3. 一次只用一个设计系统。两个 skill 的设计哲学不同（baoyu = 完整流程/deck-stage/asset管理，apple = 手感/弹簧/材质），强行融合反而失去各自的优势

## 已安装设计相关 Skill

| Skill | 定位 | 特色 |
|------|------|------|
| baoyu-design | 完整设计方法论 | 项目文件夹、deck-stage、asset record、PPTX 导出 |
| apple-design | Apple 交互手感 | 17 条原则、弹簧物理、毛玻璃材质、可打断性 |
| emil-design-eng | UI 打磨哲学 | 组件设计、动画决策、不可见细节 |
| review-animations | 动画审查 | 对标 craft bar |
| improve-animations | 动画改善 | 优先审计 + 实现计划 |
| find-animation-opportunities | 动画发现 | 扫描代码找缺失动画机会 |
| animation-vocabulary | 动画术语 | 模糊描述 → 精确术语 |

## 字体选择铁律

- baoyu-design：用系统自带字体或 skill 推荐的 DM Sans / Newsreader
- apple-design：用 -apple-system 栈（SF Pro + PingFang SC）
- 不要混用 Albert Sans（来自 Zara，不属于任何 skill）
