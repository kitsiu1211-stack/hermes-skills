# 客户名别名与 C360 搜索

## 问题

`client_list.json` 中的客户名可能与 C360 中存储的搜索关键词不同。例如：

- `client_list.json` 存的是「深圳机智连接科技有限公司」
- 但 C360 商机记录全部关联在**主体名**「深圳普洛德智能科技有限公司」下
- 用「机智连接」搜 `lark-c360 search all` 能命中 account，但商机可能不完整
- 用「普洛德」搜能拿到全部 5 条商机

## 解决方案：`c360_search` 字段

`client_list.json` 新增可选字段 `c360_search`，用于指定 C360 API 的搜索关键词：

```json
{
  "name": "深圳机智连接科技有限公司",
  "alias": "机智连接/PLAUD/普洛德",
  "c360_search": "普洛德",
  "industry": "录音设备",
  "scale": 500
}
```

| 字段 | 用途 |
|------|------|
| `name` | 显示名 + 字幕关键词匹配 |
| `alias` | 字幕关键词匹配（`/` 分隔多项） |
| `c360_search` | C360 API 搜索用关键词（覆盖 name 默认值） |

### poll-v5 中的使用逻辑

1. `_load_clients` 将 `name` + `alias` 各项 + `c360_search` 全部加入 `KNOWN_CLIENTS` 用于字幕匹配
2. `check_client_mention` 匹配到客户后，查 `client_list.json` 获取 `c360_search` 值
3. `c360_lookup` 使用 `c360_search`（若存在）作为 `--keyword`，否则用匹配到的 name
