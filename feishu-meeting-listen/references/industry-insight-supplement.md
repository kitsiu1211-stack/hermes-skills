# 行业文档 Insight 补充

## 场景

用户被 @ 要求补充行业文档中的老客洞察/行业分析，Agent 需要：

1. 通读整个专项文档（可能 160KB+，跨越多次双周会）
2. 结合 Agent 旁听过的客户会议知识
3. 输出结构化补充建议

## 分析框架

### Step 1: 定位用户被 @ 的具体任务

在文档中搜索用户名找到待办项，理解期望产出。

### Step 2: 识别文档薄弱环节

典型 gap：
- **老客洞察只有数字没有解读**（如 AI 消耗比例 → 需要补充信号分析）
- **老客名单很久没更新**（上次盘点在 4 月，7 月的新动态没写进去）
- **增购 pipeline 只列了 ARR 没写进展**
- **样板间/闭门活动分散在各次会议中，没有汇总**

### Step 3: 输出结构

按以下维度组织：

| 维度 | 内容 |
|------|------|
| 老客分层画像 | 类型分布（品牌/ODM/上游）+ ARR 梯队 + 战略价值 |
| 增购/升级 Pipeline | 从历史会议中提取在途商机 + 最新状态 |
| AI 消耗深度信号 | 不只列比例，给出解读（消耗过高=扩容信号，消耗低=渗透不足） |
| 样板间/活动进展 | 跨会议汇总 + 时间线 |
| 行业趋势 Insight | 从会议对话中提炼的模式（如品牌出海、AI焦虑但无预算） |
| 风险/盲区 | 新客断层、竞品动态、付费能力不足 |

### Step 4: 交付格式

- 先给用户看结构化分析摘要（在聊天中）
- 问用户「要我直接写进飞书文档吗？还是先跟 XX 对齐？」
- 写文档时用 `lark-cli docs` 相关命令
