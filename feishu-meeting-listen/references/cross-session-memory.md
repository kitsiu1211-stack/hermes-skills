# 跨会话双向记忆

## 问题

Hermes 在不同飞书话题/群聊中各自开独立 Session。话题里的对话和主频道里的对话互不相通，导致：
- 在话题里不知道主频道刚聊了什么
- 在主频道不知道话题里刚做了什么决策
- 用户感知：「你的记忆好像有问题，太割裂了」

## 解决方案

**回任何消息前，先 `session_search` 双向拉通。不是等用户抱怨了才搜，而是每次回复前都主动搜。**

```
收到用户消息（任何渠道）
  ├─ session_search(query=当前话题关键词, sort="newest") → 查同话题近期
  ├─ session_search(query=当前话题关键词) → 查主频道近期
  └─ 融合两个 source 的结论 → 在此基础上回复
```

**执行规则**：
- 在话题里回消息 → 同时搜主频道里的相关内容
- 在主频道回消息 → 同时搜话题里的相关内容
- 搜什么关键词：从用户消息中提取核心名词（如「AI 商机」「分享」「Karpathy」）
- **自己判断相关度，不在回复里念经「我查了 XX 个 session」**
- **触发时机：每次回复前。不是「感觉不对劲再搜」，是「默认搜一遍再回」**

## 适用场景

- AI 商机/行业分析/客户讨论 → 今天在不同话题反复聊，必须拉通
- 任何跨话题持续讨论的主题
- 用户明确说「我们之前聊过这个」

## 存储层对比

| 层 | 作用 | 跨度 |
|----|------|------|
| Session | 单话题上下文 | 当前话题 |
| session_search | 主动跨 Session 捞数据 | 所有历史 |
| Memory | 持久化摘要 | 跨所有 Session |

Memory 是我挑过的摘要（少），session_search 是全量但需主动搜。两者结合 = 完整记忆。
