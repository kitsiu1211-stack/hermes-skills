# AI 商机盘点框架

从多场 CSM 商机盘点会议中提取客户 AI 购买状态、打标签、产出跨行业对比分析的方法论。

## 数据来源

- 与各行业 CSM 逐一过客户列表的会议妙记
- 会议结构：① 已购 9.9 万以上客户 → 买的原因 ② Q3 增购/首购机会
- 每场 10-35 分钟，妙记下载后用 `read_file` 读取转写

## 标签体系

### 一、购买状态 × 动作

| 标签 | 含义 | 动作 |
|------|------|------|
| `🔴 首购-推9.9` | 未购客户 | Q3 推 9.9 万首购 |
| `🟢 加速消耗-正常` | 已购，消耗正常/超量 | 自然增购 + 产出案例 → 复制 |
| `🟡 加速消耗-需CSM` | 已购，消耗低/有问题 | CSM 重点投入推消耗 |
| `🟢 案例-标杆` | 已购大包，用量好 | 产出案例，标杆复制 |

### 二、推不动客户分层

| 标签 | 根因 | 动作 | 优先级 |
|------|------|------|:--:|
| `⚫ 低优-无决策人` | CEO 不关注 / 无 AI 负责人 | 放弃，不浪费时间 | P3 |
| `🔵 高层建联-教育` | 有 CIO/CTO 但认知不够 | AI 架构师活动 / 伟纯研修班 | P2 |
| `🟣 高层建联-触达` | 建联不上高层 | 协同 PR / 市场资源触达 | P1 |

### 三、行业特有阻因

不同行业客户不买 AI 的原因不同，不能一刀切：

| 行业 | 阻因形态 | 本质 |
|------|---------|------|
| 大制造 | AB 面恐惧（内部真实 vs 对外披露不一致） | 行业早期，方向未定 |
| 大消费 | CTO 离职潮 / CSM 服务断层 | 组织不稳定，买了没人接 |
| 消费电子 | 自己搭 agent → 不需要飞书 AI | 能力溢出 |
| 企业服务 | 豆包竞争 / 铺货电商没场景 | 替代品 + 行业属性 |

## 跨行业对比维度

| 维度 | 对比内容 |
|------|---------|
| Q3 商机总数 | 各 CSM 的商机数量 |
| 最大单客户 | 金额最大的客户及状态 |
| 核心驱动场景 | 什么场景最有效（VOC / 门店 / 知识问答 / aily 高管分身…） |
| 知识问答→增购路径 | 是否有成功案例 |
| 行业特有阻因 | 不买的特殊原因 |
| 高频词 | 该 CSM 嘴里最高频的词 |

## 核心洞察框架

### "一箭双雕"逻辑

```
推 AI 商机 → 自然需要见高层（CIO/CTO/CEO）
    ↓
见高层本身就是 RM KPI（1 号位建联率）
    ↓
一个动作同时完成：高层建联 + AI 增购
```

### 增购路径

```
知识问答（入门船票）
    → 消耗起来了 → 自然增购（最稳）
    → aily 智能伙伴 → 部门级知识库 → 场景深挖
    → 高管数字分身 → 大制造特有路径
    → 效率先锋大赛 → 跑出场景 → 增购
```

### 消耗驱动 vs 场景驱动

- **消耗驱动增购**：最稳，不需要挖场景。客户自己用量上来，自然要买更多包
- **场景驱动增购**：需要 CSM 投入，先让客户点亮智能伙伴 → 用量催生场景 → 场景催生大包

## 输出格式

发给领导汇报时用飞书交互卡片，结构：

```
1. 五行业商机全景（数量 + 典型案例表）
2. 商机标签分层（按标签统计分布）
3. 横向发现（跨行业洞察 3-5 条）
4. 后续动作（点亮率 + 场景深挖 + 一箭双雕）
```

## 注意事项

- 先盘完所有 CSM 再统一输出，不逐个行业出
- 会议标题 ≠ 字幕中提到的所有客户——不要从字幕猜，标题写谁就是谁的会
- "随飞书上线捆绑购买"的客户：有包但没场景，需要服务驱动
- 国央企客户基本无商机，快速跳过不深究
