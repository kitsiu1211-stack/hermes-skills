# 客户名单检测机制

v4 脚本从 `~/.hermes/data/client_list.json` **动态加载**客户名单，不再使用本文档中的硬编码列表。本文档仅作参考说明。

## 加载逻辑

```
client_list.json
  → Python 提取所有客户全称
  → 自动生成简称变体（去掉「有限公司」「股份有限公司」等后缀）
  → 对含「科技」的客户名，提取前半段作为额外简称
  → 注入 bash 数组 KNOWN_CLIENTS
```

## 匹配规则

- 每条新字幕/聊天消息与 `KNOWN_CLIENTS` 逐项做子串匹配
- 命中任意一个客户名即触发告警（发送到 Agent 协作群）
- 同时尝试调用 C360 CLI 查询客户信息（当前 C360 CLI 不可用，接口已预留）

## 示例

客户全称「深圳感臻智能股份有限公司」→ 字典包含：
- `深圳感臻智能股份有限公司`
- `深圳感臻智能`
- `感臻智能`
- `深圳感臻`

消息中出现「感臻」即命中。

## 维护

客户名单在 `client_list.json` 中维护。新增客户后，守护进程需重启才能生效（下次 `bash poll-v4.sh` 启动时重新加载）。
