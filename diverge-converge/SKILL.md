# 扩散收敛法（Diverge-Converge）

面对陌生复杂问题时的通用提问框架。AI 扮演思维合伙人，通过扩散→收敛的双相引擎，把模糊想法逼近成完整可实施方案。

## 核心原理

**双钻模型（Double Diamond）的 Agent 版**：

```
读懂想法 → [框架摊开（可选）] → [扩散 ↔ 收敛]（循环，重心滑向收敛）→ 检测闭合 → 结晶手稿
```

- **扩散**：铺开地图——暴露 unknown unknowns、暗礁、隐性约束、被忽略的变量
- **收敛**：逐轮锁定决策——每个分叉选定方向后不可逆，直到完整路径浮现

扩散追广度，收敛追确定性。一散一收，完整方案从不完整想法中孵化。

## 触发条件

以下信号触发此 skill：
- 用户说「扩散收敛」「diverge-converge」「先扩散后收敛」
- 用户说「帮我把这个想法展开」「看清全貌」「有哪些我没考虑到的点」
- 任何模糊、复杂、没有现成答案的开放课题
- 用户想在动手前把想法聊清楚

## 对话协议

### 1. 读懂想法

先让用户把想法说出来。不要急着分析——先完整听完，用自己的话复述一遍，确认理解正确。

### 1.5 框架摊开（可选）

触发词：`先不收敛` `摊开所有框架` `左右脑互博` `让我看看有哪些思考角度`。

此阶段委托给 `grill-with-docs` 的「框架摊开模式」（左右脑互博）执行：
- 把所有互相矛盾的思考框架全部摊开，不给予推荐
- 每个框架标注 ✅ 成立条件 / ❌ 避而不谈 / ⚠️ 失效边界
- 用户自己选择框架后，进入扩散收敛循环

**不需要时直接跳过，进入步骤 2。**

### 2. 扩散收敛循环

**此阶段完全委托给 `grill-with-docs` skill 执行**——grill-with-docs 是 diverge-converge 和 loop-engineering-pipeline 的共用追问内核。

核心纪律：
- 一次一个问题，带置信度和推荐方案
- 对比表格辅助决策
- 逐轮锁定，维护决策台账
- 答完一维再进下一维

扩散阶段以「铺开地图」为主（暴露暗礁、未知约束），收敛阶段以「锁定决策」为主（逐项不可逆选定）。两个阶段之间没有硬切换——重心从扩散平滑滑向收敛。

### 3. 检测闭合

当以下条件满足时，扩散收敛完成：
- 所有关键分叉已锁定
- 没有遗留的「以后再说」
- 一条可走通的完整路径已浮现
- 用户可以清晰描述这个方案给别人听

### 4. 结晶手稿

用户说出收口句（如「形成可实施文档」「结晶」「保存方案」）时，将整段对话沉淀为手稿。

**手稿包含**：
- 问题陈述
- 所有已锁定的决策 + 依据
- 所有被否决的方案 + 否决理由
- 技术/业务约束
- 实施路径（MVP 优先）
- 留给未来的口子（显式标记的待定项）

## 与 loop-engineering-pipeline 的关系

| | diverge-converge | loop-engineering-pipeline |
|---|---|---|
| **颗粒度** | 方案/框架级 | 代码/实现级 |
| **交付物** | 决策手稿 | 可运行代码 + tickets + handoff |
| **适用** | 研究、商业、产品、写作、规划 | 软件开发、bug 修复、功能迭代 |
| **下游** | 聊出来是软件方案 → 切 loop | 从 PRD 开始的全链路工程实现 |
| **追问内核** | `grill-with-docs` | `grill-with-docs` |

**两者共用 `grill-with-docs` 作为追问内核。**

## 关键原则

- **永远不要丢掉自己的判断力**——AI 铺地图，你选方向
- **扩散阶段不承诺路径**——看不见地形就选路 = 局部优化，撞墙时改动成本极高
- **手稿是起点，不是终点**——框架搭好后，挑存疑的关键点深入验证
